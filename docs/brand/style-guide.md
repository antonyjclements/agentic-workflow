# Augmented Workflow Brand Style Guide

Augmented Workflow is the repo-native operating model for teams that want human intent, AI execution, and reliable outcomes to compound across real software projects.

## Brand Core

### Positioning

Augmented Workflow gives product and engineering teams a durable memory layer for AI-assisted work. It keeps product intent, standards, decisions, corrections, and session context versioned with the code, so every agent starts from the same source of truth.

### Tagline

Human intent. AI execution. Reliable outcomes.

### Short Description

Augmented Workflow is a lightweight, repo-native workflow for teams using coding agents on long-lived codebases.

### Long Description

Augmented Workflow installs portable instructions, skills, and documentation structures into a repository so humans and AI agents can share durable context. Living specs, indexed standards, immutable decisions, correction learnings, and session memory help teams preserve why work happened, not just what changed.

## Logo System

### Primary Mark

Use the AW monogram as the primary visual identifier. The mark uses a continuous angular form that reads as both the initials AW and a connected workflow path.

### Logo Lockup

Use the full lockup when introducing the brand:

- AW monogram
- Augmented Workflow wordmark
- Tagline: Human intent. AI execution. Reliable outcomes.

Use the monogram alone for app icons, favicons, social avatars, small badges, and places where the name appears nearby.

Use `augmented-workflow-mark.svg` when the mark needs to scale cleanly. Use the split JPG assets for previews, raster-only placements, and compositions that need the supplied icon-tile treatments. Keep the source sheet only as a reference or for recreating derived assets.

### Clear Space

Keep clear space around the mark equal to at least the height of the monogram stroke. Do not place text, icons, borders, or page edges inside that space.

### Minimum Size

Keep the full lockup large enough for the wordmark and tagline to remain legible. When the tagline becomes too small to read, use the wordmark-only lockup or the monogram.

### Backgrounds

Approved treatments:

- Dark monogram on white or light neutral backgrounds.
- White monogram on near-black backgrounds.
- Navy-to-violet gradient monogram on white, light neutral, or black icon tiles.

Avoid low-contrast placements, busy image backgrounds, and effects that blur the AW letterforms.

## Color

Use a restrained technical palette with a high-contrast neutral base and a violet accent.

| Role | Color | Hex |
| --- | --- | --- |
| Ink | Deep navy-black | `#111827` |
| Surface | White | `#FFFFFF` |
| Soft surface | Cool off-white | `#F7F8FB` |
| Muted text | Graphite | `#5F6673` |
| Accent | Workflow violet | `#7C6CFF` |
| Accent dark | Indigo | `#39406D` |

Use violet as an accent for marks, links, focus states, and selected states. Do not let violet dominate full pages.

## Typography

Use clean, modern sans-serif typography. Prefer system fonts for product surfaces and a precise geometric sans for marketing materials when available.

- Product/UI: system sans-serif stack.
- Marketing headings: geometric sans-serif, uppercase only for short labels or logo-adjacent text.
- Body copy: sentence case, direct wording, generous line height.

Avoid condensed display faces, decorative type, and heavy letter spacing outside brand lockups.

## Voice

The brand voice is precise, practical, and outcome-oriented.

Write like:

- "Make the repo the source of truth for agent work."
- "Capture decisions once; reuse them every session."
- "Turn human corrections into durable agent guidance."

Avoid:

- Hype about replacing people.
- Vague autonomy claims.
- Mascots or playful metaphors.
- Overpromising deterministic results from probabilistic systems.

### Writing Discipline

Agents and humans should write in specific claims, not generic AI-marketing patterns.

Avoid common LLM tropes:

- "It's not X, it's Y."
- "More than just..."
- "In today's fast-paced world..."
- "Unlock the power of..."
- "Seamlessly integrates..."
- "Supercharge your workflow..."
- "Revolutionize how teams..."
- "Game-changing..."
- "Harness the power of AI..."
- "Say goodbye to X and hello to Y."

Prefer direct, grounded wording:

- Name the artifact, behavior, or outcome.
- State who uses it and when.
- Explain the constraint or tradeoff plainly.
- Use concrete nouns: specs, decisions, standards, learnings, gates, traces, repos.
- Remove filler adjectives unless they describe a verifiable property.

## Messaging Pillars

### Human Intent

Humans set product direction, constraints, standards, and acceptance criteria. Augmented Workflow preserves that intent where agents can read it.

### AI Execution

Agents do the implementation work with clear routing, current context, and project-specific standards instead of relying on one-off prompts.

### Reliable Outcomes

Specs, decisions, review gates, traceability, and correction learnings make repeated AI-assisted work more predictable across sessions and contributors.

## Do And Do Not

Do:

- Use "Augmented Workflow" on first reference.
- Use "AW" only after the full name is clear.
- Keep the mark crisp, high contrast, and unaltered.
- Pair the brand with concrete workflow artifacts and outcomes.

Do not:

- Stretch, rotate, outline, or recolor the mark outside the approved palette.
- Put the tagline in tiny placements where it cannot be read.
- Describe Augmented Workflow as an AI coding tool, IDE, or project management system.
- Use retired brand names in current marketing, docs, installer output, or badges.
